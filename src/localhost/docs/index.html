About packages

All packages have names in next format (BNF-like):

<Package name> ::= nYume<Version>_(<Target platform>_bin|src).zip
<Target platform> ::= win32|linux

You should know, what package to download. Remember simple rules:
  • always download only latest version,
  • download linux-binaries if you use any Linux platform,
  • download win32-binaries if you use any Windows platform,
  • download src-package if you want to modify and/or to compile server.

I can't compile FreeBSD package of nYume, because I don't have that system. But if you will compile nYume on FreeBSD, please, contact me and I will add FreeBSD-package.

Installation from binary packages

Windows platforms

Copy archived package in folder you want nYume to be installed in. For example, in C:\WebServer.

Unpack ZIP archive. There will be folder called nYume. You will have path like C:\WebServer\nYume.

Run file install.bat and follow instructions.

Linux platforms

Copy archived package in folder you want nYume to be installed in. For example, in /usr/WebServer.

Unpack ZIP archive. There will be folder called nYume. You will have path like /usr/WebServer/nYume.

Run file install.sh and follow instructions.

Installation from source code

You can install nYume from source code. You need to get cross-platform source code and compiler FPC 2.0.2 or later. Compilation is not depends on platform, you just need to compile file nYume.pp using command

fpc nYume

You can compile russian or english version. By default, english version will be compiled. To compile russion version of nYume, you need to remove line {$define lang_en} from file lang.inc.

After compilation you should install nYume as it is described above.

Configuration

Just after installation

Modify parameters ip and port in file config.cfg. Then modify parameters default and localhost in file vhost.cfg. Read more about those parameters in the following.

Configuration files

All parameters are in following format (BNF-like):

<Parameter name>=<Parameter Value>

config.cfg

ip
IP-address of network interface. Can be used Internet or Ethernet IPv4 address. Use 127.0.0.1 to use on local mashine.

port
Port to use with Server (1 — 65535). Standart HTTP port is 80. But if you already have running HTTP-server on port 80, you need to choose other port. To avoid conflicts, use ports greater than 1024.

deletelog
If true, then log-file will be cleared every time you run server. If false, then log-file will be just added by new information.

logfile
Filename of log-file.

error404
Filename of page to show if error 404 detected.

error403
Filename of page to show if error 403 detected.

index
Default page in directory. If user types address with terminating slash (www.site.com/fold/), this string will be added to the end of request (www.site.com/fold/index.html).

mime.cfg

This file defines headers, which server send to client. Every parameter consists of file extention and MIME-type. When client send request for a file with defined extention, server replies with header, which tells browser about file type.

If client requests about file with undefined extention, server will use MIME-type, defined with parameter default.

vhost.cfg

This file defines directories for sites. Every site is located at their own directory. You need to define, which directory belongs to requested hostname.

For example, line

test1=E:\MyProjects\fpc\nYume/test1

means that site http://test1/ is located at E:\MyProjects\fpc\nYume/test1 directory.

Note, that domain www, as well as other subdomain needs their own parameter.

Parameter default defines which directory should be used, when client doesn't request certain domain name or when there is no record for requested domain. For example, you can define directory C:\nYume\localhost for host localhost (http://localhost/) and directory C:\nYume\default for undefined host (http://127.0.0.1/). You can use any hostname, which can be in Host field of HTTP header.

blacklist.cfg

This is the black list. On each line can be located one IPv4 address. Client with this IP address will not be abble to get any file from server. The only page he will see is Error 403 page.

For example, file with content

140.12.15.20
45.1.87.29

means that client with referred above IP-addresses cannot access any site located on nYume server.

Work with program

Running nYume

Windows platforms

To run nYume you need just execute file nYume.exe.

Linux platforms

To run nYume you need just execute file nYume under root. Make sure, that file nYume has permissions, enough to execute it.

Stopping nYume

Type quit or q straight in nYume console and press Enter.

Changing configuration

First of all you need stop nYume. Then you can change configuration files. After all changes have been made, just run nYume server again.

Viewing logs

Log file consists of user requests. There are headers which clients send to server, client IP-addresse and date/time of connections.

Note, that in russian version of nYume log file will be in ASCII (cp866) charset.

Adding virtual hosts

Add directory which will be used to allocate new site. It can be any directory of your system, even if it is already used by some site. Then change configuration of nYume as it is described above. You need to change file vhost.cfg.

If you work under Linux, make sure that host directory has right permissions.
